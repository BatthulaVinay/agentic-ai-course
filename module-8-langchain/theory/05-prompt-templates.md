# Introduction

In previous modules, we learned that the quality of an AI application's output depends heavily on the quality of its prompt. A well-designed prompt can significantly improve accuracy, consistency, and reliability.

When building simple applications, developers often write prompts as plain Python strings:

```python
topic = "Machine Learning"

prompt = f"Explain {topic} in simple words."
```

While this works for small examples, it quickly becomes difficult to maintain in larger applications. Prompts become longer, variables increase, and different tasks require different prompt structures.

LangChain solves this problem using **Prompt Templates**.

Prompt Templates allow developers to create reusable prompt structures where only the variable values change while the overall prompt remains consistent.

---

# Why Prompt Templates Exist

Imagine building a chatbot that answers questions about different topics.

Without Prompt Templates:

```python
prompt = f"Explain {topic} in simple words."
```

Every prompt is manually created.

Now imagine hundreds of prompts across multiple files.

Problems quickly appear:

* Duplicate code
* Difficult maintenance
* Inconsistent prompt formatting
* Harder debugging
* Difficult prompt updates

Prompt Templates eliminate these issues.

---

# What is a Prompt Template?

A Prompt Template is a reusable prompt containing placeholders.

Instead of writing the complete prompt every time, developers define the structure once and supply values later.

Example:

```python
"Explain {topic} in simple words."
```

Here:

```
{topic}
```

is a placeholder.

During execution:

```python
topic = "Artificial Intelligence"
```

becomes:

```
Explain Artificial Intelligence in simple words.
```

---

# Prompt Template Architecture

```text
Variables
     │
     ▼
Prompt Template
     │
     ▼
Formatted Prompt
     │
     ▼
LLM
     │
     ▼
Response
```

The Prompt Template sits between the application and the LLM, ensuring that every prompt follows a consistent structure.

---

# Types of Prompt Templates

LangChain provides different prompt abstractions depending on the use case.

### PromptTemplate

Used for simple text prompts.

Example:

```
Summarize {article}.
```

---

### ChatPromptTemplate

Designed for chat models.

Instead of a single string, prompts are composed of multiple messages.

This is the most commonly used Prompt Template in modern LangChain applications.

---

# Understanding Chat Models

Modern LLMs such as GPT-4, Claude, and Gemini do not simply receive one large string.

Instead, they receive a sequence of messages.

Example:

```text
System Message
↓

Human Message
↓

AI Message

↓

Human Message

↓

Model Response
```

This conversational structure allows the model to better understand roles and context.

---

# Message Types

LangChain provides several message classes.

---

## System Message

The System Message defines the behavior of the AI.

Examples:

```
You are a helpful assistant.

You are a Python expert.

You are a medical advisor.
```

The System Message acts as permanent instructions throughout the conversation.

Architecture:

```text
System Message
      │
      ▼
Controls Model Behaviour
```

---

## Human Message

Represents the user's input.

Example:

```
Explain Machine Learning.
```

Architecture:

```text
User
 │
 ▼
Human Message
```

---

## AI Message

Represents a previous response generated by the model.

Useful for maintaining conversation history.

Example:

```
Machine Learning is a branch of Artificial Intelligence...
```

---

## MessagesPlaceholder

Sometimes conversations contain dynamic history.

Instead of manually inserting previous messages, LangChain provides `MessagesPlaceholder`.

Architecture:

```text
Conversation History
       │
       ▼
MessagesPlaceholder
       │
       ▼
Chat Prompt
```

This is commonly used with memory and chat applications.

---

# Complete Prompt Architecture

A typical chat application follows this structure:

```text
System Message

↓

Conversation History

↓

Current User Message

↓

LLM

↓

Response
```

Every request sent to the model follows this sequence.

---

# Dynamic Prompt Construction

Instead of writing:

```
Explain Machine Learning.
```

developers create templates.

Example:

```
Explain {topic} for a {audience}.
```

Input:

```
topic = Neural Networks

audience = Beginner
```

Final Prompt:

```
Explain Neural Networks for a Beginner.
```

This makes prompts reusable across many requests.

---

# Why Prompt Templates Matter

Imagine changing one instruction.

Without templates:

```
Update 50 prompts.
```

With Prompt Templates:

```
Update one template.
```

Every application automatically benefits.

This significantly improves maintainability.

---

# Best Practices

* Separate prompt logic from business logic.
* Keep prompts reusable.
* Use variables instead of hardcoded values.
* Use System Messages for permanent instructions.
* Keep Human Messages focused on user requests.
* Store long prompts in dedicated files for large projects.
* Version prompts when working in production.

---

# Common Mistakes

### Hardcoding prompts

Bad:

```python
prompt = "Explain AI."
```

Better:

```python
prompt = "Explain {topic}."
```

---

### Mixing instructions and variables

Keep instructions stable.

Only variables should change.

---

### Very long prompts

Instead of one enormous prompt, split responsibilities across multiple prompt templates.

# Interview Questions

### What is a Prompt Template?

A reusable prompt containing placeholders that are filled dynamically during execution.

---

### Why use Prompt Templates instead of strings?

They improve reusability, maintainability, and consistency.

---

### What is the purpose of a System Message?

It defines the model's behavior and acts as permanent instructions.

---

### What is the difference between Human Message and AI Message?

Human Message represents user input, while AI Message represents previous model responses.

---

### What is MessagesPlaceholder?

A placeholder that dynamically inserts conversation history into a prompt.
